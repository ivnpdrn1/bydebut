Si byDebut evoluciona como la has planteado (lector → IA → versiones → comunidad → monetización), tiene muchas más fuentes de ingreso que una editorial tradicional o una plataforma de lectura.

# Etapa 1: Hackathon / MVP

Objetivo: validar uso, no maximizar ingresos.

| Fuente                         | Precio     | Escalabilidad |
| ------------------------------ | ---------- | ------------- |
| Suscripción lector             | $5–10/mes  | Alta          |
| Compra de créditos adicionales | $1–20      | Alta          |
| Suscripción autor              | $10–30/mes | Media         |
| Biblioteca Premium             | $5–15/mes  | Alta          |

Ejemplo:

* 1,000 usuarios
* $8/mes promedio

Ingresos:

**$8,000/mes**
≈ **$96,000/año**

# Etapa 8: SaaS para Autores y Editoriales

Es el modelo que probablemente más guste a inversionistas.

byDebut como:

**"GitHub para historias"**

Planes:

| Cliente           | Precio      |
| ----------------- | ----------- |
| Autor             | $20/mes     |
| Equipo creativo   | $99/mes     |
| Editorial pequeña | $499/mes    |
| Editorial grande  | $2,000+/mes |

Con solo:

* 500 autores
* 50 editoriales

podrías superar fácilmente:

**$250,000–500,000/año**

sin depender del éxito de ningún libro.

# Escenario más atractivo para inversionistas

No es:

❌ vender libros

No es:

❌ cobrar por generación de IA

Es:

✅ Plataforma SaaS

*

✅ Marketplace

*

✅ Comisiones por licencias

*

✅ Datos de audiencia narrativa

Ese modelo puede convertir a byDebut en una empresa de software con márgenes mucho mayores que una editorial tradicional.

Para el Hackathon Gemini, yo presentaría la visión económica así:

**Fase 1:** Suscripciones de lectores.

**Fase 2:** Marketplace de versiones.

**Fase 3:** SaaS para autores y editoriales.

**Fase 4:** Sistema operativo global para evolución de historias.

Esa progresión muestra una ruta creíble desde un MVP construido en Lovable hasta una plataforma multimillonaria si logra adopción.
