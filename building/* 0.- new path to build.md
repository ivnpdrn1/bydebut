# Mi decisión como CTO

Y aquí voy a tomar una decisión firme.

## NO vamos a empezar por Claude Code.

Ni por React.

Ni por Next.js.

### Vamos a empezar por Supabase.

¿Por qué?

Porque el **backend define el contrato del producto**.

Si el backend está bien diseñado:

* Claude Code será mucho más rápido.
* El frontend tendrá menos cambios.
* La IA tendrá un contexto consistente.
* El Story Graph tendrá una base sólida.

En cambio, si comenzamos por el frontend, es muy probable que acabemos modificando pantallas y flujos porque el modelo de datos todavía no está claro.
