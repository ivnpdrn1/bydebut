## Part 2 — Information Architecture & Navigation System

---

# 1. Purpose

This section defines how byDebut is organized from the user's perspective.

It answers:

* Where does each user enter?
* What does each user see first?
* What are the main portals?
* What are the primary navigation paths?
* Which screens belong to the MVP?
* Which screens belong to Beta?
* Which screens belong to the long-term roadmap?

This is the foundation for the React / Next.js frontend.

---

# 2. Platform Structure

byDebut has **two primary experiences**.

```text
byDebut
│
├── Business Portal
│   └── For authors, publishers, editors, publicists, rights holders
│
└── Story Portal
    └── For readers and Reader-Creators entering a Story Passport
```

This distinction is essential.

The Business Portal is about managing Story Universes.

The Story Portal is about entering one.

# 5. Business Portal Pages

## 5.1 Universe Console

The main business dashboard.

Shows:

* Active Story Passports
* Reader activation
* New Lineages
* Top Reader-Creators
* Story Gravity
* Creative Momentum
* Pending approvals
* Recent activity

Primary CTA:

```text
Create New Story Passport
```

---

## 5.2 Story Passports

List of all Story Passports owned or managed by the user.

Actions:

* View Passport
* Edit Passport
* Manage rights
* Download QR
* Launch campaign
* View lineages
* View analytics

---

## 5.3 Catalog

Shows all works controlled by the user or organization.

Each item includes:

* Title
* Author
* Rights status
* Passport status
* Reader activation
* Lineage count
* Universe Health

## 5.4 Campaigns

Used by authors, publishers and publicists.

* End date
* Eligible participants

* Reward
* Analytics

## 7.2 Explore Lineages

## 7.3 Continue the Story

This is the most important creative flow.

The screen should ask:

```text
What would you change?
```

Options:

* Change the ending
* Save a character
* Change the protagonist
* Add romance
* Add mystery
* Change the time period
* Make it cinematic
* Create a children's version
* Write my own idea

Then:

* AI Creative Planner creates a plan
* User approves
* AI Writer generates draft
* AI Reviewer reviews it
* User edits
* User publishes as a Lineage

# 9. MVP Navigation Scope

The Hackathon MVP should include only:

## Public

* Home
* How It Works
* Pricing
* Sign In
* Get Started

## Business Portal

* Universe Console
* Create Story Passport
* Story Passport Details
* Catalog
* Basic Analytics

## Story Portal

* Story Passport
* Continue the Story
* Explore Lineages
* My Creations

## Admin

* Minimal internal view

Everything else can be marked as Beta.

# 12. Core UX Rule

No user should ever be more than one click away from the next meaningful action.

Examples:

Author:

```text
Create Story Passport
```

Publisher:

```text
View Portfolio Growth
```

Reader:

```text
Continue This Story
```

Reader-Creator:

```text
Publish My Lineage
```

---

# 13. Navigation Priority

The frontend should never expose all modules equally.

Priority is role-based.

Authors see:

```text
Story Passports
Analytics
Campaigns
Community
```

Readers see:

```text
Story Passport
Continue
Lineages
Community
```

Publishers see:

```text
Catalog
Portfolio Analytics
Campaigns
Rights
```
