
# 5. La capa más importante: Service Layer

Las Edge Functions no deben contener toda la lógica mezclada.

Cada Edge Function debe seguir este patrón:

```text
Recibir request
Validar datos
Identificar usuario
Llamar servicio interno
Guardar resultado
Responder al FrontEnd
```

La lógica reutilizable debe estar organizada mentalmente así:

```text
SemanticContextBuilder
GeminiClient
EmbeddingService
SemanticSearchService
VersionGenerationService
ReviewService
CreditService
LogService
```

En Supabase Dashboard, al trabajar desde web, probablemente pegarás cada función completa en su propio `index.ts`.

Pero internamente el código debe estar organizado con secciones.

# 6. Semantic Context Builder

Esta es la pieza más importante del backend.

Su trabajo no es llamar a Gemini.

Su trabajo es construir el texto que Gemini debe recibir.

## Regla principal

Gemini no sabe qué columnas existen.

Gemini solo vectoriza o analiza lo que tú le envías.

Por eso el backend debe hacer esto:

```text
Leer tabla Supabase
   ↓
Seleccionar campos semánticos
   ↓
Construir semantic_text
   ↓
Enviar a Gemini
```

# 10. Gemini Client

Todas las llamadas a Gemini deben estar centralizadas.

Habrá tres tipos de llamadas:

```text
generateText()
generateReview()
generateEmbedding()
```

## Gemini para embeddings

Se usará para convertir `semantic_text` en vector.

```text
semantic_text
   ↓
Gemini text-embedding-004
   ↓
vector
   ↓
public.embeddings
```

## Gemini para generación

Se usará para crear nuevas versiones literarias.

```text
book/chapter/scene + variables + mode
   ↓
Gemini generation
   ↓
story_versions
```

## Gemini para review

Se usará para analizar una versión generada.

```text
story_version
   ↓
Gemini reviewer
   ↓
ai_reviews
```

# 11. Embedding Service

Responsabilidad:

```text
Recibe entity_type + entity_id
Busca el registro en Supabase
Construye semantic_text
Llama a Gemini embedding
Guarda en embeddings
Devuelve embedding_id
```

Esta lógica vive dentro de la Edge Function:

```text
create-embedding
```

---

# 12. Semantic Search Service

Responsabilidad:

```text
Recibe pregunta del usuario
Convierte pregunta en embedding
Ejecuta match_embeddings()
Recupera entidades relacionadas
Devuelve resultados al FrontEnd
```

Esta lógica vive dentro de:

```text
semantic-search
```

---

# 13. Version Generation Service

Responsabilidad:

```text
Recibe book_id/chapter_id/scene_id
Recibe variables narrativas
Recibe generation_mode
Construye prompt
Llama a Gemini
Guarda story_version
Guarda version_variables
Resta crédito
Crea embedding de la nueva versión
Devuelve resultado
```

Esta lógica vive dentro de:

```text
generate-version
```

---

# 14. Review Service

Responsabilidad:

```text
Recibe version_id
Lee story_version
Construye prompt de review
Llama a Gemini
Guarda ai_reviews
Devuelve análisis
```

Esta lógica vive dentro de:

```text
review-version
```
